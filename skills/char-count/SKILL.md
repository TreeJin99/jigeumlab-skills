# 글자수 체크 (char-count) v0.1.0

한국 정부지원사업 양식·신청서·사업계획서 본문이 글자수 제한(예: "1,000자 이내")을 가질 때 **자동 발동**하여 정확한 글자수를 측정하고 OVER/OK 판정을 보고한다. 시각 추정은 절대 금지하며 반드시 카운트 도구로 측정한다.

> **핵심 가치**: Claude는 한국어 본문 길이를 시각으로 추정하면 평균 100~300자 오차가 발생한다 (실측 사례: "약 800자"로 추정한 본문이 실제 1,083자). 양식 1,000자 제한 환경에서 이 오차는 신청서 반려 사유다. 본 스킬은 시각 추정을 도구 카운트로 강제 교체한다.

## Workflow

### Phase 1 — 제한 인식 (Limit Detection)

다음 중 **1개라도 충족**하면 본 스킬을 발동한다:

- 양식 PDF·HWP·이미지·텍스트에 "글자수 제한: N자" 패턴 등장
- 사용자 프롬프트에 "X자 이내", "X자 안에", "X자 제한", "X자 이하" 명시
- 본문 작성·검토·압축 요청 시 양식상 글자수 제한이 알려진 경우 (작업 컨텍스트 기억)
- 사용자가 "글자수 체크" / "자수 카운트" / "X자 맞나" 직접 요청

각 제한 N과 본문 식별자(섹션명·항목명)를 메모하고 Phase 2로 진행.

### Phase 2 — 본문 추출 (Body Extraction)

- 본문이 마크다운 파일에 있는 경우: 헤딩·양식 텍스트·표·코드블록·인용문·메타데이터 제외하고 사용자가 채운 본문 텍스트만 추출
- 양식 자체 표기("글자수 제한: 1,000자", 섹션 제목, 안내 문구)는 카운트 **제외**
- 추출 도구: PowerShell `[regex]::Match` 또는 Bash `awk`/`sed`

### Phase 3 — 카운트 (Count)

- **표준 도구**: PowerShell `.Length` (Windows 환경 우선) → Python `len()` → Bash `wc -m` (한국어 정확도 ↓로 후순위)
- **공백 포함** 카운트가 한컴오피스·정부 양식 기본
- `.Trim()` 적용 — 본문 전후 공백 제외, 본문 내부 공백·줄바꿈은 포함

PowerShell example:

```powershell
$text = [System.IO.File]::ReadAllText($path, [System.Text.Encoding]::UTF8)
if ($text -match $pattern) {
    $body = $Matches[1].Trim()
    $len = $body.Length
}
```

Python example:

```python
import re
with open(path, 'r', encoding='utf-8') as f:
    text = f.read()
body = re.search(pattern, text, re.DOTALL).group(1).strip()
print(len(body))
```

### Phase 4 — OVER/OK 판정 + 마진 보고 (Verdict)

- `$len > 한도` → ❌ OVER, 초과량 표시
- `$len ≤ 한도` → ✅ OK, 마진(여유 자수) 표시
- 다중 본문 일괄 검사 시 표 형태로 출력

OVER 발생 시 **자동 압축 금지** — 사용자에게 OVER 안내 + 압축 옵션 제시. 사용자가 명시적으로 "압축해줘" 요청한 후에만 압축 진행 (Constraints §자동 압축 금지).

## Output Schema

```
[글자수 체크 — 한도 N자]

| 항목 | 자수 | 마진 | 판정 |
|---|---|---|---|
| {섹션명1} | {len1} | +{margin} or -{over} | ✅ OK / ❌ OVER |
| {섹션명2} | {len2} | +{margin} or -{over} | ✅ OK / ❌ OVER |

(요약) 전체 K개 항목 중 J개 OK / L개 OVER
```

OVER 항목이 있으면 마지막에 안내:

```
❗ {OVER항목명} {over}자 초과 — 압축이 필요합니다.
   압축 진행할까요? (네 / 부분만 압축 / 사용자 직접 수정)
```

## Constraints (Capability Uplift — 영구 강제)

- **시각 추정 절대 금지**: "약 800자 정도일 듯", "1,000자 정도 되는 것 같다" 같은 시각·감각 추정 금지. 반드시 카운트 도구로 측정한 후 응답
- **공백 포함 표준**: 한컴오피스·한국 정부 양식 기본 = 공백 포함. 공백 제외 카운트는 사용자 별도 요청 시에만 추가 표시
- **본문만 카운트**: 양식 텍스트("글자수 제한: 1,000자", 섹션 제목, 안내 문구)는 카운트 **제외**. 사용자가 채운 본문(빈칸 응답)만 카운트
- **자동 압축 금지**: OVER 발생 시 자동으로 압축 X. 사용자에게 OVER 알림 + 압축 옵션 제시. 사용자가 "압축해줘" 명시 요청 시에만 압축 진행
- **카운트 도구 우선순위**: Windows = PowerShell `.Length` > Python `len()` > Bash `wc -m`. 한국어는 `wc -m`이 일부 멀티바이트 문자에서 부정확하므로 후순위
- **다중 본문 일괄 처리**: 여러 본문이 같은 한도(예: 양식 1,000자 본문 10개)를 가지면 한 번의 도구 호출로 일괄 측정 + 표 형태 출력
- **본문 작성 직후 자동 카운트**: 본 스킬이 발동된 컨텍스트에서 본문을 작성했다면 작성 직후 자동으로 카운트 + OK/OVER 보고. 사용자 명시 요청 없어도 발동

## Anti-Triggers (이 스킬은 호출하지 않음)

- **코드 길이 추정**: "이 함수 너무 길어", "리팩토링해서 줄여줘", "PR 너무 큰 거 같아" — 양식 도메인 아님. 코드 복잡도·줄 수는 별개
- **SNS 글자수**: "트위터 280자 맞춰줘", "인스타 캡션 길이 체크" — 본 스킬은 한국 정부지원사업 양식 전용. 단, 사용자가 명시적으로 "이 도구 써서 SNS 글자수도 체크해줘"라고 요청하면 카운트 도구는 적용 가능
- **일반 문장 길이 감각**: "이 문장 짧아?", "이 문단 너무 길지?" — 정확 카운트 의도 아닌 감각 평가. 본 스킬은 정량 측정용
- **번역·영작문 단어 수**: "이 영어 글 단어 수 알려줘" — 본 스킬은 한국어·공백 포함 글자수 카운트가 표준. 영어 단어 수(word count)는 별개 도메인
- **Claude 응답 자체 길이 평가**: "이 답변 너무 길지?" — Claude 출력 길이는 양식 도메인 아님
- **카톡·메신저 메시지 길이**: "이 카톡 너무 길지?" — 양식 글자수 제한이 아닌 가독성 판단
