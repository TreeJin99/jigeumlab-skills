# 비교표 모드 (Comparison Table Mode)

> osint-deepsearch의 실행 모드 중 하나. **N개 대상을 동일한 축(필드)으로 병렬 조사해 비교표를 생산**한다. 경쟁사 비교·시장 플레이어 비교·솔루션 벤치마크·사업계획서 경쟁사 분석표용.
>
> **Phase 3 교차검증·A/B/C/D 등급·TRIANGULATE·Phase 5 저장은 본 SKILL.md와 100% 동일하게 적용**한다. 이 문서는 "조사 구조(items × fields)와 항목별 병렬 실행"만 바꾼다. 검증·저장 규칙은 안 바꾼다.

## 언제 이 모드
- "경쟁사 비교표", "N개 비교", "플레이어 비교", "벤치마크표 만들어줘", 사업계획서 "경쟁사 분석(표)", "솔루션 비교"
- 비교 대상이 **2개 이상이고 같은 축으로 정렬**해야 할 때. 단일 주제 심층은 본 SKILL.md 단일/하네스 모드로.

## Step 1 — items(대상) 정의 · HITL 게이트
- 비교 대상을 사용자와 합의해 확정. **5~12개 권장**(너무 많으면 라운드 상한 초과).
- 누락 후보를 1회 제안(예: "이 셋 외에 ○○도 비교군에 넣을까요?") 후 사용자 승인.

## Step 2 — fields(비교 축) 정의 · HITL 게이트
각 필드 = `이름 · 정의 · detail · 주 출처유형`. detail은 조사 깊이를 셀 단위로 통제한다.

| detail | 셀 내용 |
|---|---|
| `brief` | 한 줄 값/수치 (예: "월 3.9만원") |
| `moderate` | 값 + 1줄 근거 |
| `detailed` | 값 + 근거 + 맥락 2~3줄 |

**사업계획서 경쟁사 표준 필드 (기본 제안, 사용자가 가감):**

| 필드 | 정의 | detail | 주 출처유형 |
|---|---|---|---|
| 핵심 기능/제품 | 무엇을 파는가 | moderate | 공식 홈페이지·보도자료 |
| 가격/요금제 | 단가·구독료 | brief | 공식·입점몰 |
| 타겟 고객 | 누구를 노리나 | brief | 뉴스·IR |
| 차별점 | 우리 대비 강·약점 | detailed | 종합 |
| 규모(매출·고용) | 사업 규모 | moderate | DART·잡코리아 |
| 비고(투자·인증 등) | 특이사항 | brief | 뉴스·공시 |

→ 필드 확정 전 사용자 승인.

## Step 3 — 항목별 병렬 조사

### 실행 방식 분기
| 조건 | 방식 |
|---|---|
| 대상 ≤4 또는 간단 | 메인 세션에서 항목별 순차 조사 |
| 대상 ≥5 또는 직인용 수치 다수 | **`Workflow` fan-out — 항목 1개 = 에이전트 1개 병렬** (패턴은 `references/harness-mode.md` 차용) |

### 각 항목(에이전트) 조사 규칙
1. 해당 item을 **모든 field에 대해** 조사 — 필드 detail level만큼만.
2. **Gemini 그라운딩 1회 필수**: `node scripts/gemini_research.js "<대상>" "<필드별 검색 프롬프트>"`
3. 한국 시장 대상이면 화이트리스트 `site:` 1회(`references/trusted-domains.md`).
4. **각 셀에 출처번호 `[N]` + 신뢰도 등급 `(A~D)`** 부착. 본 SKILL.md 등급 기준 그대로.
5. 확인 불가 셀 = **`[확인필요]`** — 그럴듯한 수치 임의 생성 절대 금지.
6. 직인용 수치(매출·점유율·시장규모)는 **TRIANGULATE**(독립 출처 3개) 적용 → 미달 시 A등급 금지.

### 한국 시장 소스 라우팅 (필드유형별)
| 필드유형 | 1차 소스 |
|---|---|
| 시장규모·통계 | KOSIS·통계청·한국은행·업종별 협회 |
| 기업 재무·고용 | DART 전자공시·잡코리아/사람인·벤처확인기업·중소기업현황 |
| 제품·가격 | 공식 홈페이지·네이버쇼핑·입점몰·보도자료 |
| 정책·지원 이력 | K-Startup·기업마당(bizinfo)·중기부 공고 |
| 트렌드·평판 | 주요 언론·전문 매체·리뷰 |

> 기본은 **한국 시장 소스 우선**. 다만 비교 대상·시장이 중국(중국 기업, 중국 시장 진출 분석 등)이면 **중국 시장 소스도 포함**한다 — 국가로 막지 않는다.
> 제외 대상은 국적이 아니라 **리서치와 무관한 개발자 Q&A 커뮤니티 라우팅**(StackOverflow·GitHub 이슈 등). Deep-Research-skills 원본의 개발자용 검색 모듈은 컨설팅·시장 리서치에 무용하므로 차용하지 않는다.

### 재개(resume) — 대규모일 때만
항목별 결과를 `{output_dir}/{item-slug}.md`로 즉시 저장하면, 중단 후 **미완 항목만** 재개할 수 있다. 이미 파일이 있는 항목은 재조사하지 않는다. (Deep-Research-skills의 resume-dedup 차용)

## Step 4 — 검증 게이트
- **빈 셀 0건** — 미확인은 전부 `[확인필요]`로 채움.
- 모든 셀에 등급 부착. **D등급 셀은 표 하단 주석으로 모아** "단일/약한 출처" 경고.
- 직인용 수치 TRIANGULATE 미달분은 본 SKILL.md `## 단일 출처 의존 항목`에 분리.

## Step 5 — 출력
1. **비교표** = items(행) × fields(열). 각 셀: `값 [출처번호] (등급)`.
2. `## 비교 인사이트` — 격차·포지셔닝 빈틈·우리 진입 기회 3~5불릿.
3. `## 단일 출처 의존 항목` / `## Missing Pieces` (본 SKILL.md 동일).
4. **Phase 5 저장 동일**: `report.md`(비교표 포함) + `sources/`(A·B등급 셀 근거만 스냅샷).

### 비교표 출력 예시
```
| 대상 | 핵심 기능 | 가격 | 규모(매출) | 차별점 |
|---|---|---|---|---|
| A사 | … [1](A) | 월 4만원 [2](B) | 30억(2024) [3](A) | … |
| B사 | … [4](B) | 비공개 [확인필요] | [확인필요] | … |
> D/확인필요: B사 가격·규모 — 공식 미공개, 추정 금지
```
