# PDF·양식 읽기 규칙

<HARD-GATE>
PDF 파일은 반드시 이미지(비주얼) 모드로 읽을 것.
텍스트 추출 모드는 표 구조·칸 배치·체크박스·셀 병합 등 레이아웃 정보를 소실시킴.
</HARD-GATE>

## Workflow

### Phase 1 — PDF 수신 확인

- 파일 경로 또는 첨부 확인
- 파일 유형 판별: 공고문 / 신청서 양식 / 일반 보고서 / 기타
- 용도 확인: 내용 요약 / 양식 작성 보조 / 레이아웃 분석

### Phase 2 — 이미지 모드 읽기 (필수)

- Read 도구의 이미지(비주얼) 모드로 PDF 열기
- 1페이지씩 순서대로 스캔하며 레이아웃 파악
- 표, 체크박스, 2열 구조, 병합 셀 위치를 원본 기준으로 확인

### Phase 3 — 결과 구조화

- 요청 용도에 맞게 출력:
  - **내용 요약**: 섹션별 핵심 내용 불릿
  - **양식 작성 보조**: 원본 표 구조를 마크다운 표로 변환 (구조 최대 보존)
  - **레이아웃 분석**: 2열 짝 구조, 체크박스 위치, 필수 기재란 식별

## Output Schema

```
## PDF 읽기 결과

파일명: {파일명}
파일 유형: {공고문 | 신청서 양식 | 보고서 | 기타}
읽기 모드: 이미지(비주얼)

### 주요 내용
{요청 용도별 구조화 출력}

### 레이아웃 주의사항 (양식 PDF의 경우)
- {마크다운 변환 시 유의할 구조적 특이사항}
```

## Constraints

- **이미지 모드 강제**: 텍스트 추출 모드 사용 절대 금지. 특히 정부지원사업 공고문·서식·양식 PDF는 표 레이아웃 자체가 핵심 정보
- **원본 구조 보존**: 마크다운 표로 변환할 때 원본 2열 짝 구조, 병합 셀, 체크박스 위치를 최대한 반영
- **임의 재구성 금지**: 가독성을 위해 테이블을 분리·병합·재배치하지 않음

## Anti-Triggers

- **텍스트 파일 읽기**: `.txt`, `.md`, `.csv`, `.json` 등 일반 텍스트 파일 — 이미지 모드 불필요
- **코딩·디버깅**: 파이썬 에러 수정, 코드 리뷰 등
- **PDF 미제공 상태**: 파일 없이 PDF 관련 일반 질문 ("PDF를 어떻게 만들어?")
