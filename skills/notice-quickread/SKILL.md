# 공고문 빠른 분석 (v2.0.0)

/kickoff 전 단계에서 정부지원사업 공고문 PDF를 빠르게 읽고 핵심만 추출하는 **분석 전용** 스킬. 폴더 생성·노션 등록·슬랙 전송 등 셋업 동작은 일절 수행하지 않으며, 마감·평가배점·자격·결격·서류·지금랩 해석을 9-블록 템플릿으로 출력한다.

## Workflow

### Phase 1 — 공고 PDF 경로 확인 (Path Resolution)

- 유저가 PDF 경로를 직접 제공 → 즉시 Phase 2로 진행
- 경로 부재 → `../03. 공용 리소스/지원사업 공고/` 하위 탐색
  - 매칭 후보 1건 → 자동 선택 후 진행
  - 매칭 2건 이상 → 후보 목록 제시 + 선택 대기 (추측 매칭 금지)
  - 매칭 0건 → Constraints §부재 fallback에 따라 안내 후 종료

### Phase 2 — PDF 읽기 (PDF Image-Mode Read)

- Constraints §PDF 이미지 모드 강제 규칙에 따라 **반드시 이미지(비주얼) 모드로 Read** (`.claude/rules/pdf-reading.md` §1 정합)
- 긴 공고는 핵심 페이지(사업개요·자격요건·평가기준·제출서류) 우선 읽기
- 표 구조·체크박스 위치·셀 병합 등 레이아웃 정보 보존

### Phase 3 — 핵심 항목 추출 (Key Element Extraction)

Output Schema의 9개 블록을 채울 수 있는 항목을 추출:
- 마감일시 / 지원 규모 / 주관 기관
- 평가 배점 표 (마크다운 표)
- 자격 요건 (필수) / 가산점 요건 / 결격 사유
- 제출 서류 (필수·가산)
- 지금랩 해석 (평가 최우선축·주의 자격·차별화 포인트 3축 의무 도출)

### Phase 4 — 후속 안내 (Hand-off Suggestion)

Output Schema 끝에 두 위임 옵션을 안내:
- 이 클라이언트로 진행: `/kickoff {클라이언트} {담당자} / {사업명} / notice={경로}`
- 자료조사 먼저: `/research`

## Constraints (Encoded Preference — 영구 강제)

- **PDF 이미지 모드 강제**: 공고문 PDF는 반드시 이미지(비주얼) 모드로 Read. 텍스트 추출 모드는 표 구조·체크박스·셀 병합 등 레이아웃 정보를 손실시키므로 금지 (`.claude/rules/pdf-reading.md` §1 정합)
- **분석 전용 원칙**: 본 스킬은 공고 분석만 수행. 폴더 생성·노션 페이지 등록·슬랙 전송·캘린더 등록·파일 작성 절대 금지. 셋업·후속 작업은 /kickoff 등 별도 영역
- **9-블록 출력 형식 고정**: Output Schema 블록(요약 헤더·마감·지원·주관·평가배점·자격·가산·결격·서류·해석·후속안내) 순서·이모지 마커 고정. 블록 누락·재배치 금지. 정보 없는 블록은 "(공고에 명시 없음)" 표기로 빈 블록 유지
- **핵심 페이지 우선 읽기**: 긴 공고는 사업개요·자격요건·평가기준·제출서류 페이지 우선. 전체 페이지 순차 읽기는 토큰 낭비
- **지금랩 해석 의무 (3축)**: 평가 최우선축(배점 가장 높은 항목)·주의할 자격(까다로운 요건)·차별화 포인트(비어있는 포지션) 3개 항목 의무 도출. 추출 실패 시 "(분석 보강 필요)" 명시
- **클라이언트명 미부착 원칙**: 본 스킬은 공고 자체 분석. 출력에 임의로 클라이언트·담당자명을 부착하지 않음. 클라이언트 매칭은 /kickoff 영역
- **탐색 폴더 분리**: 경로 부재 시 `../03. 공용 리소스/지원사업 공고/` 하위만 탐색. 클라이언트 폴더(`02. 클라이언트/`) 탐색 금지 (역할 분리)
- **부재 fallback**: 공용 리소스 폴더에서 매칭 0건 시 "공고 PDF를 찾을 수 없습니다. 경로를 직접 제공하거나 `03. 공용 리소스/지원사업 공고/`에 PDF를 먼저 배치해 주세요"로 안내 후 종료. 임의 다운로드·검색 시도 금지

## Anti-Triggers (이 작업은 호출하지 않음)

- **비공고 PDF 일반 요약**: "이 논문 요약해줘", "이 책 핵심만", "이 매뉴얼 요약해줘", "이 보고서 정리" — 정부지원사업 공고가 아닌 PDF는 본 스킬 영역 외
- **이미 진행 중인 사업 작업 재개**: "X 클라이언트 사업계획서 이어서", "X 어디까지 했지?" → client-context 영역
- **신규 클라이언트·사업 셋업**: "이 공고로 새 사업 시작해줘", "/kickoff", "신규 등록해줘" — 본 스킬은 분석 전용, 셋업은 /kickoff 영역
- **공고문 생성·작성 요청**: "공고문 만들어줘", "우리 회사 버전으로 공고 작성", "공모 안내문 작성" — 본 스킬은 읽기 전용 분석
- **이미 분석된 공고의 단건 조회**: "예시기업 사업 마감일이 언제였지?" — 이미 진행 중인 사업의 단건 조회는 client-context의 공고_요약.md 영역
- **PDF 외 형식 공고 분석**: hwp·docx·xlsx 공고 양식 — 본 스킬은 PDF 전용. 다른 형식은 별도 처리
