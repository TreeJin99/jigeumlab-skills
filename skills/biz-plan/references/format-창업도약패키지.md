### □ 창업아이템 개요 및 사업화 계획 (요약, **2페이지 이내**)

| 행 | 내용 |
|---|---|
| 아이템명 + 범주 | 서비스명 / 카테고리 |
| 아이템 개요 | AS-IS(현재 한계) ↔ TO-BE(도약 후) 2열 비교 + 핵심 수치 |
| **중장기 사업 로드맵** | 3단계 화살표 (협약기간/중기/장기) + 단계별 핵심전략+목표 |
| **협약기간 내 사업화 로드맵** | 간트차트 (협약 시작~종료, 월별) + 주요 마일스톤 3~5개 |
| **협약기간 내 산출물 목표** | 기술고도화 / 실증·검증 / 사업화 3축 분리 표시 |

### 1. 문제인식 (Problem)

**1. 창업아이템 개발 동기(필요성) 및 현황**

**1-1. 외적동기** — 시장 수요·정책 변화 기반
- 시장 규모 수치 + 성장률(CAGR)
- 구조적 미스매치(공급-수요 불일치) → 시장 공백 도출
- 3대 문제 병목 도식 (아이콘 + 수치 카드)

**1-2. 내적동기** — 대표자 현장 운영 경험
- 현장 운영하며 발견한 "구조적 한계"
- 기존 버전 한계 → 전환 결정 근거
- 플랫폼 진화 타임라인 도식 (Ver.1 → Ver.2 → Ver.3.0 목표)

**1-3. 필요성** — 시스템으로 풀어야 하는 근거
- 인력으로는 해결 불가능한 구조적 문제 명시
- 기존 대응의 한계 (현재 기술 수준) vs 도약 후 목표
- "3년간 검증된 현장 인프라 × AI 기술력 = 보유 중" 결론 강조

---

### 2. 실현가능성 (Solution)

**2-1. 창업아이템 목표시장(고객) 분석**

**2-1-1. 창업아이템 소개 및 개선과제**
- 서비스 4단계 플로우 도식 (체험→입학→관리→정착 등)
- 보유 성과 수치 3~4가지 강조 카드
- 국내·글로벌 협력기관 로고 그리드

ㅇ 핵심 차별점 — MOAT 구조 도식
- 4중 해자 동심원: 네트워크 해자 / 데이터 해자 / 기술 해자 / IP 해자
- 각 해자별 구체적 근거 + 복제 소요 기간

ㅇ 제품·서비스의 개발 및 고도화 방안
- AI 시스템 아키텍처 도식: INPUT → AI ENGINE → OUTPUT
- 핵심 모듈별 개발 방안 카드 (AS-IS 한계 / TO-BE 개발목표 / 기대효과)
- 모듈 예: AI OCR 자동검증 / 행정 워크플로우 자동화 / AI 다국어 상담 / 시스템 API 연동

ㅇ 핵심 비즈니스 모델 (BM)
- B2C/B2B/B2G 타겟별 매출 비중 도식
- 수익 모델 산출 근거 표: 수익모델/단가/목표수량/예상매출/산출근거
- 타겟별 비중 및 제공 서비스 표

## 도식화 포인트 [나노바나나 작업 대상]

- 요약: AS-IS vs TO-BE 2열 비교 박스 + 3단계 로드맵 화살표
- 협약기간 간트차트: 월별 바 + 마일스톤 다이아몬드
- 협약기간 산출물: 3축 카드 (기술고도화/실증검증/사업화) 색상 구분
- 문제인식: 구조적 미스매치 → 공백 → 기회 3단 흐름 도식
- MOAT: 4중 해자 동심원 다이어그램 (복제 기간 표시)
- BM 도식: B2C/B2B/B2G/해외 4채널 + 비중 % 원형
