## 문서 기본 형식

- 표지 없음. **□ 신청현황 → □ 일반현황** 이중 구조로 시작
- 페이지 번호: `- N -` 형식, 하단 중앙
- 섹션 구분: 숫자. 제목 (배너 박스) + □ 소항목
- 글머리: `ㅇ` (중항목), `-` (세부), `■` (소섹션)

### □ 창업아이템 개요(요약) — PSST 4개 셀 통합 표

| 행 | 내용 |
|---|---|
| 명 칭 + 범 주 | 서비스명 / 카테고리 (AI/Ed-Tech/빅데이터 등) |
| **아이템 개요** | 서비스 소개 도식 + 핵심 수치 4개 (핵심기술/수익모델/검증현황/고객혜택) |
| **문제 인식 (Problem)** | 시장 성장 + 구조적 장벽 도식 + 해결책 강조 박스 |
| **실현 가능성 (Solution)** | STEP1 완료도 % 카드 4개 + STEP2 딥테크 엔진 카드 3개 + AS-IS→TO-BE |
| **성장전략 (Scale-up)** | 수익모델 도식(채널별비중%) + 3단계 시장확보전략 + 경쟁사비교 + BM산출근거 + 타겟별비중표 |
| **팀 구성 (Team)** | 대표자 이력 표 + 팀 역량·네트워크 표 (현조직/신규채용/MOU/협력기관) |
| 이미지 | 실제 서비스 화면 + 서비스 화면 디자인 (캡션 필수) |

### 1. 문제인식 (Problem) — 창업아이템의 고도화 필요성

**1-1. 외적동기** — 시장 수요·정책 변화
- 시장 규모(조 원) + CAGR 그래프
- "Why Now?" 3가지 정책·환경 변화 카드
- 구조적 진입 장벽: 기존 솔루션 한계 %로 표기

**1-2. 내적동기** — 현장 운영 경험 기반
- 현장에서 발견한 "구조적 한계" (기존 버전 운영하며 확인)
- 기술 한계 명확화: "기존 방법으로는 불가"
- 플랫폼 진화 타임라인 도식 (Ver.1→Ver.2→Ver.N 목표)

**1-3. 필요성** — 시스템으로 풀어야 하는 근거
- 인력으로는 해결 불가능한 구조적 문제
- "3년간 검증된 현장 인프라 × AI 기술력 = 보유 중" 결론 강조

---

### 2. 실현가능성 (Solution) — 창업아이템의 고도화 계획

**선행연구 개발현황 및 개선 계획**

ㅇ STEP 1 | 기반 인프라 (선행연구) 완료 현황
- 4개 완료도 카드: 완료율% + 항목명 + 실적 수치
  - 예: 100% 저작권해결기술 / 95% B2C 플랫폼 / 100% 파일럿 / 30% 트랜스포머 설계

ㅇ STEP 2 | 딥테크 핵심 R&D (추가 개발)
- 3개 딥테크 엔진 카드 (각 TRL 단계 명시)
  - 카드 구성: 엔진명 + 기반 기술 + 목표 수치 + TRL N~M + 현재 진행 상황

ㅇ 최종 목표: AS-IS(현재) → TO-BE(딥테크 고도화 후) 비교 도식

ㅇ 경쟁사 비교 분석
- 포지셔닝 결론 강조 박스: "자사만의 통합 강점"
- 경쟁사 비교표: 기술기능/딥테크AI/데이터 3개 축 + ×/○/△/✓ 표기

ㅇ 핵심 차별점 — MOAT 4중 해자 구조
- 4중 해자 동심원: IP 해자 / 데이터 해자 / 기술 해자 / 네트워크 해자
- 각 해자별 구체적 근거 + 복제 소요 기간
- "N중 해자 = 경쟁사가 넘을 수 없는 N중 해자" 결론

ㅇ 제품·서비스의 개발 및 고도화 방안
- AI 시스템 아키텍처: INPUT → AI ENGINE → OUTPUT 3단계
- 핵심 모듈별 개발 방안 (각 모듈 카드):
  - AS-IS 한계 / TO-BE 개발목표 / 6단계 플로우 도식 / 기대효과

## Pre-TIPS 고유 체크리스트

- [ ] **□ 신청현황 별도 섹션** (□ 일반현황 앞에 위치)
- [ ] **투자유치 금액 + 투자자 유형** 명시 (TIPS 연계 필수 요건)
- [ ] **[가점] 모두의아이디어 수상 여부** 체크
- [ ] 요약 표에 PSST 4개 셀 모두 포함
- [ ] STEP1 선행연구 완료도 % 카드 (4개)
- [ ] STEP2 딥테크 핵심 R&D 카드 (3개, TRL 명시)
- [ ] MOAT 4중 해자 동심원 다이어그램
- [ ] AI 시스템 아키텍처 INPUT→ENGINE→OUTPUT 도식
- [ ] 모듈별 6단계 개발 플로우 도식

## 분량 가이드

- 신청현황 + 일반현황: 1쪽
- 요약표: 2~3쪽
- Problem: 3~4쪽 (외적/내적/필요성)
- Solution: 4~5쪽 (딥테크 엔진 상세)
- Scale-up + Team: 3~4쪽 (공고문 확인 필요)
- **총 13~17쪽 추정** (공고문 확인 필요)

## 도식화 포인트 [나노바나나 작업 대상]

- 요약 실현가능성 셀: STEP1 완료도 % 카드 4개 + STEP2 딥테크 엔진 카드 3개
- STEP1/STEP2 진행 상황: % 원형 게이지 + TRL 바
- MOAT: 4중 해자 동심원 (복제 기간 표시)
- AI 아키텍처: INPUT → ENGINE → OUTPUT + 기술 스택 표기
- 모듈별 플로우: 6단계 박스 화살표 (단계별 개선율 %)
- 경쟁사 비교: 3개 축 구분 그리드 (자사 열 하이라이트)
