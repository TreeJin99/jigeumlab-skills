> ⚠️ 2025년 합격작 기준 패턴. 당해 공고문과 반드시 대조 후 사용.

## 문서 기본 형식

- 표지 없음. 제목 박스(굵은 테두리) + 일반현황부터 바로 시작
- 페이지 번호: `- N -` 형식, 하단 중앙
- 섹션 구분: `□` 기호 + 제목 (본문 챕터는 숫자. 제목 배너 박스)
- 본문 글머리: `ㅇ` (중간 항목), `-` (세부 항목), `•` (글머리 강조)

---

## 섹션 구조 (순서 고정)

### □ 일반현황 (표)
| 항목 | 내용 |
|---|---|
| 창업아이템명 | (2줄 이내) |
| 산출물 (협약기간 내 목표) | 단계별 산출물 나열 |
| 직업 (직장명 기재 불가) | 대학생 / 무직 / 직종만 기재 |
| 기업(예정)명 | |

**팀 구성 현황 (대표자 본인 제외)**
| 순번 | 직위 | 담당 업무 | 보유 역량(경력 및 학력 등) | 구성 상태 |
- 구성 상태: "완료" 또는 "예정 (YY.MM)"

---

### □ 창업 아이템 개요(요약) (큰 표 1개)

| 행 | 내용 |
|---|---|
| 명 칭 + 범 주 | 가로 2칸: 서비스명 / 카테고리 |
| 창업 아이템 개요 | 도식(서비스 플로우 or 제품 이미지) + 핵심 키워드 + 3~5줄 특징 |
| 문제 인식 [Problem] | 문제점-해결방안 대비표 + 시각자료(그래프, 사진) |
| 실현 가능성 [Solution] | 실험/개발 진행 사항 + 지산권 + 일정 표 |
| 성장전략 [Scale-up] | 경쟁사 비교표 + TAM/SAM/SOM + 비즈니스 모델 |
| 팀 구성 [Team] | 대표 프로필 + 업무 파트너 |
| 이미지 | 제품/서비스 실제 사진 (캡션 필수) |

> 요약 표는 전체 문서의 "요약 슬라이드" 역할. 각 셀에 도식 필수.

---

### 1. 문제 인식 (Problem)_창업 아이템의 필요성

구성 순서:
1. **소제목 1** - 기존 시장 문제 (시각 자료 + 표)
2. **소제목 2** - 페인포인트 도출 (설문 데이터, 수치 근거 필수)
3. → 결론 1줄 (굵게 강조)

표 패턴:
```
| 문제점 | 해결방안 |
| 기존 제품의 X | 자사 제품의 Y |
```

---

### 2. 실현 가능성 [Solution]

구성 순서:
1. 프로토타입/개발 현황 (before/after 사진 포함)
2. 지식재산권 현황 표
   | 출원일 | 출원번호 | 권리 구분 | 비고 |
3. 협약 기간 내 추진 일정 (간트차트: 월별 칸 채우기)
4. 경쟁사 비교표

---

### 3. 성장전략 [Scale-up]

구성 순서:
1. TAM / SAM / SOM 표 (3행 3열: 구분/정의/규모)
2. 비즈니스 모델 (수익 채널별 비중 도식)
3. 마케팅 전략 (단계별: 1단계 → 2단계 → 3단계)
4. 연도별 매출 목표 표 (연도 / 사업화 / 투자 / 매출 / 고용)

---

### 팀 구성 [Team]

구성 순서:
1. 대표 CEO (역할, 배경, 이력 1~3줄)
2. 업무 파트너 (전문 자문 등)

---

## 문체·서술 규칙

- **개조식** 필수: `-` 또는 `•`로 시작, 명사 종결
- 수치 근거 필수: "00명 대상 설문 결과 00% 응답"
- [대괄호] 카테고리 강조: `[시장 기회]`, `[구체적 문제]`
- 결론문은 `→` 화살표로 시작
- "혁신적", "게임체인저" 등 추상 수식어 금지
- 단위 표기: 백만원(백만원), 억원(억원) — 숫자는 아라비아 숫자

## 분량 가이드

- 일반현황: 1쪽
- 요약표: 1쪽
- Problem: 1~2쪽
- Solution: 1~2쪽
- Scale-up: 1쪽
- Team: 0.5쪽
- **총 5~8쪽 이내**

## 도식화 포인트 [나노바나나 작업 대상]

- 요약표 각 셀: 서비스 플로우 / Before-After 이미지
- 문제 인식: 문제점-해결방안 2열 비교표
- 성장전략: TAM/SAM/SOM 원형 다이어그램
- 경쟁사 비교: ○/×/△ 기호 사용 비교표
