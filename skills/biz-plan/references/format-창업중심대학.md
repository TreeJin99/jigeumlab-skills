## 문서 기본 형식

- 표지 없음. 제목 박스(회색 배경) + 일반현황부터 바로 시작
- 페이지 번호: `- N -` 형식, 하단 중앙
- 섹션 구분: 숫자. 제목 (배너 박스) + □ 소항목
- 글머리: `ㅇ` (중항목), `-` (세부), `•` (강조), `■` (소섹션 제목)

| 행 | 내용 |
|---|---|

### 1. 문제인식 (Problem)

**1-1. 창업아이템 배경 및 필요성**

구성:
- ㅇ 핵심 문장 (굵게)
  - 시장 규모 + CAGR 수치 (출처 명기)
  - 문제 확산 범위 (전 카테고리로 확산 중)
  - 기존 솔루션의 한계 (AI/플랫폼별)

- ㅇ 대표자 직접 경험 (Before → After 도식 가능)
  - 수치화된 개선 효과 (% 절감, 배 향상)

- ▶ 결론 강조 박스 (진한 배경)
  - PMF 달성 실적 수치 포함

**1-2. 창업아이템 목표시장(고객) 현황 분석**

구성:
- ㅇ 시장 분석 (글로벌/국내 규모 수치)
  - LAM/SOM/SAM/TAM 4분할 표
  - 핵심 시장 / 현재 위치 / 확장 전략 / 성장 전망

- ㅇ 목표 고객별 핵심 니즈 및 활용 시나리오 표
  | 구분 | 핵심 니즈(Pain Point) | 자사 활용 시나리오 | 기대 효과 |
  (SMB / B2B / B2G 등 타겟별)

- ㅇ 기대효과 표
  | 구분 | 현재(As-Is) | 자사(To-Be) | 개선율 |

### 2. 실현가능성 (Solution)

**2-1. 창업아이템의 개발/개선 준비현황**

■ 서비스 운영 현황 및 공인 검증

구성:
- [서비스 운영] 누적 수치 + 매출 YoY 성장률
- [지재권 확보] 특허 출원/등록 목록 (번호 + 내용)
- [공인 검증 完] 시험기관명 + 발행일 + 평가 결과

■ 서비스 아키텍처 현황
- INPUT → AI ENGINE → OUTPUT 3단계 도식
  - INPUT: 데이터 유형 목록
  - ENGINE: 핵심 기술 모듈 목록
  - OUTPUT: 산출물 유형 목록
- 기술 개발 방식 (자체/외부API/특허) 구분 표시

**2-2. 창업아이템 실현 및 구체화/고도화 방안**

ㅇ 제작 방법 및 소요기간
- 3개 핵심 개발 항목 카드 형식
  - 번호 + 항목명
  - [현재 수준] 현재 완성도 바 표시
  - [고도화 방법] 체크리스트
  - [핵심 투입] 인력/자원
  - [검증 방법] 목표 수치

- < 최종 산출물 작성(전체 사업단계) >
  | 구분 | 최종 산출물 | 수량 | 완료 일정 |
  (단기/협약기간내 + 중·장기 분리)

ㅇ 경쟁력 확보 방안
- 포지셔닝 맵 (2×2 축: AI자동생성↔수동작성 / 서식특화↔범용)
- 경쟁사 비교표
  | 구분 | 경쟁사A | 경쟁사B | 경쟁사C | 자사 |
  | 핵심 항목들 | ×/○/△/◎ |
  - 처리시간, 완성도 등 정량적 항목 포함
  - 결론: 자사만의 통합 강점 1줄 강조

ㅇ 기술보호 방안
- 4계층 동심원 다이어그램: 지식재산권 → 기술적 보안 → 관리적 보안 → 물리적 보안
- 각 계층별 세부 조치 목록

### 3. 성장전략 (Scale-up)

**3-1. 창업아이템 비즈니스 모델**

ㅇ 비즈니스 모델 도식
- 타겟별(SMB/B2B/B2G) → 핵심 제품 → 가치 제공 흐름
- 요금제 테이블 (Free / Basic / Pro / Enterprise)

ㅇ 핵심 비즈니스 지표 (Unit Economics)
| 지표 | ARPU | 유료전환율 | LTV | LTV/CAC | 영업이익률 |
- 수치 + 달성 근거 한 줄씩

ㅇ 고객 전환 퍼널 현황
| 단계 | 지표 | 수치 |
(유입/체험/전환/확장/제휴)

**3-2. 창업아이템 시장 진입 등 사업화 전략**

ㅇ 타겟 시장: B2C → B2B → B2G 3단계 확장 전략
- 단계별 타임라인 도식 (현재/하반기/이후)
  - 각 단계: 대상 / 전략 / 독표 / 파트너

ㅇ 각 타겟군별 진입 전략 (번호 항목)
- (1) B2C 진입 전략: 커뮤니티 + Freemium 모델
  - 대표자 커뮤니티 운영 실적 + ✓ 근거
- (2) B2B 진입 전략: 제휴처 레퍼런스 기반
  - 현재 진행 중인 제휴 + ✓ 차별화 요소
- (3) B2G 진입 전략: 공공조달 + 시범사업
  - 구체적 사업명 + 추진 일정

## 문체·서술 규칙

- 개조식 + 수치 근거 필수
- `■` (소섹션), `ㅇ` (중항목), `-` (세부 항목), `•` (강조)
- 검증된 PMF 수치 반드시 포함
- 기술보호 방안 구체적 기술 (특허번호, 등록일)
- 단위: 억원/백만원 혼용 (1억 미만은 백만원)

## 분량 가이드

- 일반현황 + 요약: 2쪽
- Problem: 2~3쪽
- Solution: 3~4쪽
- Scale-up: 3~4쪽
- **총 10~14쪽**

## 도식화 포인트 [나노바나나 작업 대상]

- 서비스 플로우: INPUT → ENGINE → OUTPUT 3단계
- 포지셔닝 맵: 2×2 축 매트릭스 (자사 위치 강조)
- 경쟁사 비교표: ×/○/△/◎ 기호 표 (자사 열 강조)
- 기술보호 4계층: 동심원 다이어그램
- B2C→B2B→B2G 단계 타임라인
- 비즈니스 모델: 요금제 테이블 (Free/Basic/Pro/Enterprise)
