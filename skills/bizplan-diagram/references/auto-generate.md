# 나노바나나 자동 생성 루프 (gemini_image.js)

> 프롬프트를 텍스트로만 주지 않고, `scripts/gemini_image.js`로 **직접 생성·확인·재생성**까지 수행.
> 과거엔 "AI Studio 수동 붙여넣기" 가정이었으나, 자동 파이프라인이 있으므로 끝까지 처리.

---

## 모델

`gemini-3.1-flash-image` (gemini_image.js 기본값) — 한글 정확·빠름·발표 품질.
구형 `gemini-2.5-flash-image`는 한글 깨짐 — 사용 금지.

---

## 절차

### 1. 프롬프트 파일 작성

긴 한글 프롬프트는 셸 이스케이프가 깨지므로 **파일로 저장 후 `--file`** 로 넘김.

```
scripts/.{도식}_prompt.txt   ← 점(.) 프리픽스로 임시 파일 표시
```

내용 = [도식 본문 텍스트·레이아웃] + [style-presets.md 의 프리셋 A 또는 B 블록].

### 2. 생성 실행

```
node scripts/gemini_image.js --file=scripts/.{도식}_prompt.txt "<출력경로.png>"
```

저장 경로(클라이언트 작업): `../jigeumlab-outputs/clients/{클라이언트}/{연도}-{프로그램}/images/{도식코드}-{설명}.png`
(지금랩 자체 자료는 해당 GDrive 작업 폴더에 직접 저장)

### 3. 결과 확인 (필수)

생성된 PNG를 **Read 로 열어 시각 검수**:
- 한글 오타·깨짐 없는지
- ASCII 드라이런 레이아웃과 일치하는지
- 수치 강조·자사 부각이 의도대로인지

### 4. 재생성 루프

어긋나면 프롬프트 파일을 수정하고 2~3 반복. 보통 1~2회 내 수렴.

---

## 환경

- API 키: `.env` 의 `GEMINI_API_KEY`
- 나노바나나 3.1이 한글을 직접 정확히 렌더 — 별도 폰트·렌더 엔진 불필요

## 생성 결과가 어긋날 때

나노바나나가 위치·오타를 가끔 틀리면 폴백을 찾지 말고 **프롬프트를 더 구체화**해 재생성:
- 레이아웃을 좌표·방향 언어로 명시("정중앙", "좌상단", "세 원이 모두 겹치는 곳")
- 틀린 한글을 따옴표로 정확히 고정("PlanDoc", "유일")
- 1~2회 반복이면 대부분 수렴
