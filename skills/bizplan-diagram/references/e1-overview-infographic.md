## 레이아웃 구조

```
┌─────────────────────────────────────────────────────────────┐
│  [서비스명]              핵심 가치 1줄                       │  ← 헤더 (전체 너비)
├────────────────────────┬────────────────────────────────────┤
│  핵심 기능 (30~40%)    │  서비스 목업 또는 플로우 (60~70%) │
│                         │                                    │
│  ● 기능1 제목           │  [목업형]                         │
│    1줄 설명             │   ┌──────────┐                    │
│                         │   │ UI 화면  │                    │
│  ● 기능2 제목           │   └──────────┘                    │
│    1줄 설명             │                                    │
│                         │  [플로우형]                        │
│  ● 기능3 제목           │   입력 → 처리 → 출력              │
│    1줄 설명             │                                    │
└────────────────────────┴────────────────────────────────────┘
```

## 구성 요소 상세

### 헤더 (전체 너비)

| 요소 | 형식 |
|---|---|
| 서비스명 | Bold 최대형 |
| 핵심 가치 1줄 | Regular, 서비스명 오른쪽 또는 아래 |

### 좌컬럼 — 핵심 기능 (30~40% 너비)

- 기능 3~5개
- 각 기능: 아이콘(선택) + 기능 제목(SemiBold) + 1줄 설명(Regular)
- 수치·통계 배제 (우컬럼에서 강조)
- 기능 간격: 동일 간격

### 우컬럼 — 목업형 vs 플로우형

**목업형** (SaaS·앱·디지털 서비스, 실제 제품 있을 때):
- 리얼리스틱 UI 목업 — 실제 화면 스크린샷 스타일
- **단순 아이콘·플랫 일러스트 금지** (리얼리스틱만 허용)
- SaaS: 실제 대시보드·입력폼 (텍스트, 버튼, 필드 포함)
- 앱: 실제 모바일 화면 (상태바, 네비게이션 포함)
- 하드웨어: 포토리얼리스틱 제품 렌더링

**플로우형** (제품 없는 초기·MVP 단계 / 오프라인·물리적 프로세스):
- 아직 실제 제품이 없는 클라이언트의 가안(假案) 표현에 사용
- 입력 → 처리 → 출력 (3~4단계)
- 단계 박스 + chevron(>) 화살표
- 각 단계에 핵심 수치 1개 배치 가능
- 각 단계에 리얼리스틱 목업 배치 권장 (단순 아이콘 지양)
