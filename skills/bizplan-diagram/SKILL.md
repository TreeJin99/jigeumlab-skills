# 사업계획서 도식화 → 나노바나나 프롬프트

사업계획서 섹션 유형으로부터 **도식 패턴을 자동 선택**하고 **나노바나나(Google AI Studio) 이미지 생성 프롬프트**를 출력하는 스킬.

> 분석 기반: 실제 클라이언트 사업계획서·IR덱 8건 (B사·C사·A사·F사·G사·H사·I사·J사)

---

## 도식 패턴 라이브러리

### A. 제안배경 / 문제인식

| 코드 | 도식 유형 | 선택 기준 |
|---|---|---|
| A-1 | 넘버링 문제 카드 ①②③ | 문제 3~4개 나열 (기본) |
| A-2 | 3중 벤다이어그램 | 3가지 공백/기회가 교차하는 구조 |
| A-3 | 풀배너 결론박스 | 섹션 마무리 핵심 메시지 1줄 (슬라이드 하단 전폭) |
| A-4 | 수렴 브라켓 | 2~4개 항목 → 우측 결론+CTA 수렴 (설득 구조) |

### B. 시장분석

| 코드 | 도식 유형 | 선택 기준 |
|---|---|---|
| B-1 | TAM/SAM/SOM (반원 아크 + 정보 패널) | 시장 규모 단계적 축소 구조 |
| B-2 | 경쟁사 O/X 비교 매트릭스 | 기능 3~5개 × 경쟁사 2~4개 비교 |
| B-3 | 포지셔닝 매트릭스 2×2 | X/Y축 설정 + 경쟁사 버블 + 자사 강조 |
| B-4 | 막대그래프 + 수치강조패널 | CAGR·시장성장 차트 + 우측 핵심수치 병치 |

### C. 솔루션 / 차별성

| 코드 | 도식 유형 | 선택 기준 |
|---|---|---|
| C-1 | AS-IS → TO-BE 카드 | 변화 전후 대비 (고객군별 또는 문제→해결) |
| C-2 | 선순환 플로우 (STEP 01~04) | 기술→데이터→서비스 선순환 구조 |
| C-3 | 파트너십 허브 | 중앙 브랜드 + 4방향 파트너 기관 |
| C-4 | SWOT 2×2 사분면 | S/W/O/T 각 사분면 + 하단 전략카드 |
| C-5 | 동심원 경쟁장벽 | 중앙 브랜드 + 레이어별 진입장벽 (2~4겹) |

### D. 성장전략 / 로드맵

| 코드 | 도식 유형 | 선택 기준 |
|---|---|---|
| D-1 | STAGE 1→2→3 로드맵 | 분기/연도별 단계 성장 계획 (현재 단계 강조) |
| D-2 | 사업전후 비교표 | 현재 실적 vs 목표 수치 비교 |
| D-3 | 마일스톤 지그재그 타임라인 | 홀수/짝수 상하 교번 배치, 연도별 마일스톤 |
| D-4 | 가로 분할 비율바 | 자금구성·예산 항목별 % (파이차트 대체) |

### E. 사업개요 / 비즈니스모델

| 코드 | 도식 유형 | 선택 기준 |
|---|---|---|
| E-1 | 2컬럼 서비스 개요 인포그래픽 | 사업계획서 첫 페이지·서비스 소개 이미지 |
| E-2 | KPI 하이라이트 카드 | 핵심지표 5개 대형 수치 카드 (IR덱 첫 페이지) |

## 나노바나나 프롬프트 공통 원칙 (실제 분석 기반)

### 수치 강조 (모든 도식 공통)
핵심 수치(시장규모·달성률·매출 등)는 **카드 내 최대 크기 Bold**로 배치. 설명 텍스트는 수치 아래 소형으로 부속화.
프롬프트 지시: `"[수치]" — Pretendard Bold 최대형, 강조색` / `"[설명]" — Regular 소형, #6B7280`

### 현재 단계·자사 항목 부각
타임라인, 비교표, 로드맵에서 **현재 단계 / 자사 항목**만 강조색 배경 또는 Bold 강조. 나머지는 회색 처리.
프롬프트 지시: `"[현재단계]" 박스만 강조색 배경, 나머지 박스는 흰색+회색 테두리`

### 풀배너 결론 (A-3 / 모든 섹션 마무리)
슬라이드 하단 전폭에 강조색 배경 배너 1줄 결론 배치. 평가위원 시선 종착점.
프롬프트 지시: `슬라이드 하단 전폭에 "[결론 문장]" — 강조색 배경 배너, 흰색 텍스트 Bold`

## Workflow

### Phase 1 — 입력 수집

다음 정보를 확인. 부족하면 AskUserQuestion으로 요청:

1. **섹션 이름 또는 코드** — "제안배경", "TAM/SAM/SOM", "SWOT" 등
2. **클라이언트 핵심 정보**: 아이템명 / 업종 / 해당 섹션 핵심 수치·키워드
3. **도식 코드** — 자동 판단 가능하면 그대로, 모호하면 후보 2개 AUQ 제시
4. **용도 (제출용/발표용)** — HWP 제출본=프리셋 A(미니멀), IR·피칭·발표 슬라이드=프리셋 B(프리미엄). 모호하면 AUQ 1회. → `references/style-presets.md`

**브랜드 컬러 오버라이드 (자동)**
클라이언트명이 특정된 경우 `../jigeumlab-outputs/clients/{클라이언트}/brand.md`를 읽는다.
- `## 나노바나나 오버라이드` 섹션의 강조색·보조색을 프롬프트 공통 스타일 블록에 주입
- brand.md 없거나 클라이언트 미특정: `e1-overview-infographic.md` 업종별 기본 팔레트 사용

### Phase 2 — 내용 구조화

선택된 도식 코드 패턴에 따라 **이미지에 실제로 들어갈 텍스트 요소** 구성:
- 라벨, 수치, 키워드, 관계 구조 목록
- 미확인 수치는 `OOO` 표기

### Phase 3 — ASCII 드라이런

````
📐 드라이런 — 레이아웃 미리보기
```
─────────────────────────────────────────────────────────────
[ASCII 레이아웃. 컬러 위치는 [BLUE]텍스트, [RED]텍스트 형식]
─────────────────────────────────────────────────────────────
```
````

드라이런 아래: "위 레이아웃 맞으면 아래 프롬프트 복사해서 나노바나나에 붙여넣기 하세요."

### Phase 4 — 나노바나나 프롬프트 출력

아래 공통 스타일 블록 완전 준수. 복사 마커 포함:

```
━━━━━━━━━ ↓ 아래 코드블록만 복사 ↓ ━━━━━━━━━
[완성된 프롬프트]
━━━━━━━━━ ↑ 위 코드블록까지 복사 ↑ ━━━━━━━━━
```

코드블록 아래 검수 포인트:
- pt/px/# 스펙 값이 이미지에 찍히지 않았는지
- 카드 상단·하단·좌우에 컬러 바·띠지가 없는지
- 의도한 한국어 텍스트만 정확히 표시됐는지

### Phase 5 — 마커 삽입 (선택)

`사업계획서.md`의 `[도식 위치]` 마커 바로 아래에 드라이런 + 프롬프트 삽입.

### Phase 6 — 자동 생성 · 확인 · 반영

`gemini_image.js`로 **직접 생성→Read 확인→재생성**까지 수행. 상세 → `references/auto-generate.md`.

1. 프롬프트를 `scripts/.{도식}_prompt.txt`로 저장(긴 한글 이스케이프 회피) + 프리셋 블록 포함
2. `node scripts/gemini_image.js --file=scripts/.{도식}_prompt.txt "<출력경로.png>"`
   - 저장 경로: `../jigeumlab-outputs/clients/{클라이언트}/{연도}-{프로그램}/images/{도식코드}-{설명}.png`
3. 생성 PNG를 **Read로 시각 검수** — 한글 오타·레이아웃·수치강조 확인. 어긋나면 프롬프트 수정 후 재생성(보통 1~2회 수렴)
4. **사업계획서.md 반영**: `[도식 위치]` 마커에 `![{설명}](images/{파일명}.png)` 삽입. 드라이런·프롬프트 블록은 최종본에서 정리

## Constraints

- **이미지 생성 모델**: `gemini-3.1-flash-image` 사용(한글 정확·빠름, gemini_image.js 기본값). 구형 `gemini-2.5-flash-image`는 한글 깨짐 — 금지. AI Studio 직접 생성 시 Nano Banana(Gemini 3.1+) 선택
- **스타일 프리셋 강제**: 용도별 프리셋(제출용 A / 발표용 B) 블록을 프롬프트에 통째 포함. Pretendard 고정 → `references/style-presets.md`
- **생성은 나노바나나 3.1 전용**: 모든 도식은 `gemini_image.js`로 생성. 어긋나면 프롬프트 구체화 후 재생성(폴백 엔진 없음)
- **수치 강조 원칙**: 핵심 수치는 항상 Bold 최대형으로 지시. 수치가 곧 도식의 핵심
- **현재 단계·자사 부각**: 타임라인·비교표·로드맵에서 현재/자사만 강조, 나머지 회색
- **수치 조작 금지**: 미확인 수치 임의 생성 금지. `OOO` 또는 `(확인 필요)` 표기
- **드라이런 필수**: 프롬프트 전 반드시 ASCII 드라이런 출력
- **도식 1개씩 처리**: "전체 다" 요청 시 순서대로 하나씩 확인
- **컬러 적용 범위 명시**: 강조색이 적용되는 텍스트 요소를 프롬프트 내 명확히 한정

## Anti-Triggers

- **이미지 생성 방식**: `gemini_image.js`로 자동 생성(Phase 6). AI Studio 수동 붙여넣기도 가능하나 기본은 자동 루프
- **HWP 파일 편집**: 파일 직접 수정은 본 스킬 범위 외
- **사업계획서 본문 작성**: business-plan 스킬 영역
- **범용 도식 (사업계획서 섹션 무관)**: 조직도·서비스 아키텍처 → /infographic
- **공고문 분석**: notice-quickread 영역
- **데이터 리서치**: "시장 규모 조사해줘", "경쟁사 찾아줘" → osint-deepsearch 영역
