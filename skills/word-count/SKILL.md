# /word-count — 한국어 텍스트 자수·바이트 체크

## Workflow

### Phase 1 — 측정 대상 확인 (Input Check)

1. **텍스트 수신** — 유저가 직접 텍스트 제공 또는 현재 작업 중인 문서에서 해당 항목 추출
2. **한도 확인** — "X자 이내" / "X자 내외" / "X자 이상" 중 어느 유형인지 확인
3. **모드 판단**:
   - 단일 텍스트 → 단일 체크 출력
   - 후보 2개 이상 → 비교 표 출력

### Phase 2 — 자수·바이트 계산 (Counting)

1. **공백 포함 글자수** 계산 (한국 양식 기본): 한글·영문·숫자·공백·문장부호 각 1자
2. **UTF-8 바이트수** 계산: 한글·전각 부호 ×3, 영숫자·반각·공백 ×1
3. **한도 통과 여부** 판정: "내외" vs "이내" 해석 적용

### Phase 3 — 결과 출력 (Output)

단일 텍스트 또는 후보 비교 표로 출력. 한도 임박·초과 시 자동 제안.

---

## 사용 시점

- 사업계획서·신청서 양식의 "X자 이내·X자 내외" 항목 작성
- 슬라이드·발표자료 헤드라인·카피 자수 제한
- 유저가 "이거 몇 자야?", "X자 이내로 줄여줘", "자수 체크" 요청
- 후보 여러 개 비교 (예: "줄임 후보 3개 만들어줘")

## 자수 측정 규칙

### 1. 공백 포함 글자수 (한국 양식 기본)
- 한글·영문·숫자 1자 = 1자
- 공백 1자 = 1자
- 문장부호 (·, ・, ., ,, !, ?) 1자 = 1자
- 카운팅 기준: Python `len(text)` / JavaScript `text.length`

### 2. 바이트수 (UTF-8 기준 — HWP·웹 폼 표시용)
- 한글 1자 = **3 byte**
- 영문·숫자·반각 부호·공백 1자 = **1 byte**
- 전각 부호 (·, ・, ※) 1자 = **3 byte**
- 계산식: 한글·전각 자수 × 3 + 영숫자·반각 자수 × 1

### 3. 양식별 통상 자수 한도

| 양식 종류 | 자수 한도 | 비고 |
|---|---|---|
| 사업계획서 아이템 개요 | 30자 내외 | "내외" = 통상 ±3자 OK, 안전하게 28자 권장 |
| 사업화 과제명 | 50자 내외 | 핵심 기술·시장·아이템 압축 |
| 한 줄 요약 (피치) | 80~100자 | 발표용 |
| 슬라이드 헤드라인 | 25자 이내 | 6×6 룰 (한 줄 6어절) |
| 슬라이드 본문 | 줄당 25자 / 슬라이드당 6줄 | |
| 카카오톡 메시지 단문 | 100자 권장 | 1화면 |
| 노션 작업 제목 | 50자 권장 | 보드뷰 카드 잘림 방지 |

## 출력 포맷

### 단일 텍스트 체크

다음 양식으로 출력:

| 항목 | 값 |
|---|---|
| 텍스트 | "헬스장 설치 AI 체형·자세 분석 운동 보조 솔루션" |
| 공백 포함 글자수 | 28자 |
| 바이트수 (UTF-8) | 51 byte (한글 16 × 3 + 영숫자/공백 3) |
| 양식 한도 30자 | ✅ 통과 (2자 여유) |

### 후보 비교 (2개 이상)

| 후보 | 자수 | 바이트 | 한도 통과 | 특징 |
|---|---|---|---|---|
| A. 헬스장 설치 AI 체형·자세 분석 운동 보조 솔루션 | 28자 | 51 byte | ✅ (30자) | 체형·자세 둘 다 + "운동 보조" 유지 |
| B. 헬스장 설치 실시간 AI 자세·체형 분석 보조 솔루션 | 29자 | 53 byte | ✅ | "운동" 빠짐, "실시간" 살림 |
| C. 헬스장 설치형 실시간 AI 자세 분석 운동 솔루션 | 25자 | 47 byte | ✅ | 체형·보조 빠짐 (단순화) |

## 자동 추천 규칙

- **권장 분량**: 한도의 80~95% (여유 5~20%)
- **한도 임박** (95~100%): ⚠️ 표시 + 추가 줄임 옵션 제시
- **한도 초과**: ❌ 표시 + 줄임 후보 3개 자동 제안

## "내외" vs "이내" 해석

| 표현 | 해석 |
|---|---|
| "30자 **내외**" | 통상 ±3~5자 OK, 안전하게 한도 -2자 (28자) 권장 |
| "30자 **이내**" | 30자 초과 절대 금지, 권장 27~29자 |
| "30자 **이상**" | 최소 30자 보장, 다소 길어도 OK |

## Red Flags — STOP

| 잘못된 판단 | 올바른 기준 |
|---|---|
| "30자 내외 = 정확히 30자" | "내외" = ±3~5자 허용. 안전하게 한도 -2자 |
| 바이트수만 보고 자수 한도 충족 판단 | 한국 양식은 자수(글자) 기준이 95% — 바이트는 보조 |
| 영문 단어수(word) 카운트 | 한글은 글자(char) 기준, 영문 양식만 word count |
| 공백 빼고 카운트 | 한국 양식은 통상 공백 포함 |
| 줄바꿈·탭을 글자에서 제외 | HWP·웹 폼은 줄바꿈도 1자로 취급 |

## Common Rationalizations

| Excuse | Reality |
|---|---|
| "한 글자 정도 넘어도 OK" | 30자 한도면 31자도 시스템에서 컷되는 경우 다수 (특히 웹 폼) |
| "내외니까 35자도 OK" | 통상 ±3~5자, 10자 초과면 평가위원이 "준수 미흡"으로 판단 |
| "바이트 500까지니 자수는 무관" | 일부 양식은 자수+바이트 둘 다 한도 (예: 200자/600byte) |

## Quick Reference

- 기본: 한국 양식 = 공백 포함 글자수 기준
- 바이트는 UTF-8 기준 (한글 3, 영숫자 1)
- "내외"는 ±3~5자 / "이내"는 한도 엄수
- 후보 비교 시 표 출력 (자수·바이트·한도 통과·특징 4열)
- 한도 80~95% 권장, 100% 임박 시 ⚠️

## Constraints

- **공백 포함 기본**: 한국 양식은 공백 포함 글자수 기준. 바이트는 보조 정보로만 제공
- **"내외" 해석**: ±3~5자 허용, 안전하게 한도 -2자 권장. 10자 초과는 "준수 미흡" 경고
- **바이트 독립 판단 금지**: 바이트수만으로 한도 통과 여부를 판정하지 않음
- **자동 줄임 제안**: 한도 초과 시 줄임 후보 3개를 자수·특징 비교 표로 자동 제안

## Anti-Triggers

- **영문 word count**: 영어 텍스트의 단어수 카운트는 이 스킬 범위 외 (word는 한국어 양식과 기준 다름)
- **코딩**: Python/JS로 자수 카운터 구현 → frontend-dev 영역
- **일반 문서 분량 가이드**: "이 글 너무 길어?" 류의 정성적 평가 → 자수 한도가 명시된 경우에만 활성화
- **사업계획서 본문 작성**: 줄임 후보 채택 후 전체 섹션 작성 → business-plan 스킬 영역
